# MailSender500 📧

**MailSender500** — это Desktop-приложение на базе Spring Boot, предназначенное для безопасной массовой рассылки электронных писем с собственного почтового домена.

Приложение разработано с учетом защиты от блокировок: оно строго соблюдает интервалы отправки и суточные лимиты, сохраняя весь прогресс в локальной базе данных.

---

## ✨ Основные возможности

*   **Умная отправка**: Фиксированный интервал в 30 секунд между письмами.
*   **Контроль лимитов**: Автоматическая остановка при достижении 500 писем в сутки.
*   **Работа с EML**: Поддержка готовых файлов писем (.eml) с сохранением HTML-верстки и вложений.
*   **Импорт из TXT**: Чтение базы адресатов из текстового файла с разделителем `;`.
*   **Отказоустойчивость**: База данных H2 (файловая) хранит статус каждого письма. Если приложение закроется, оно продолжит с того же места.
*   **Desktop UI**: Простой графический интерфейс на Swing.

---

## 🛠 Технологический стек

*   **Java 22** (Amazon Corretto)
*   **Spring Boot 3.x** (Data JPA, Mail Starter)
*   **H2 Database** (Embedded)
*   **Lombok**
*   **Jakarta Mail** (для парсинга .eml)

## 🚀 Быстрый старт

### 1. Требования
*   Установленная **JDK 17** или **22**.
*   **IntelliJ IDEA** (или другая IDE).
*   Данные вашего SMTP-сервера.

### 2. Настройка почты
Откройте файл `src/main/resources/application.properties` и укажите данные вашего почтового домена.

### 3. Запуск приложения
Запустите главный класс `MailSender500Application.java`. После запуска откроется окно управления.

---

## 📖 Инструкция по использованию

1.  **Подготовка базы**: Создайте файл `list.txt`, где адреса указаны через точку с запятой:
    `test1@mail.com; test2@mail.com; test3@mail.com`
2.  **Загрузка базы**: Нажмите кнопку **"1. Выбрать базу контактов"**. Данные попадут в локальную БД.
3.  **Загрузка письма**: Подготовьте письмо в формате `.eml`. Нажмите **"2. Выбрать файл письма"**.
4.  **Старт**: Нажмите **"СТАРТ РАССЫЛКИ"**. Приложение начнет отправлять письма.
5.  **Мониторинг**: Следите за ProgressBar и счетчиком в нижней части окна.

---

## ⚠️ Важные замечания

*   **Лимиты**: Приложение не отправит более 500 писем за последние 24 часа.
*   **База данных**: Все данные хранятся в файле `mail_db.mv.db` в корневой папке проекта.
*   **Headless Mode**: В коде приложения принудительно отключен режим `headless` для работы Swing.

---

## 🛠 Возможные ошибки

*   `AuthenticationFailedException`: Проверьте логин/пароль или используйте пароль приложения.
*   `Connection Timeout`: Проверьте настройки порта (465 или 587).
